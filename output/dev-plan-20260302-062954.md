- Recommendation: Define one canonical runtime path, implement `--semantic/--refresh-cache` there first, then enforce parity tests that fail CI when sibling copies diverge.

**[CRITICAL] Ollama Is a Hard Single Point of Failure for Semantic Mode**
- Location: Phase 1 Embed function; Ollama API; Phase 2 Search flow
- Issue: Semantic retrieval depends on one local process and one model endpoint, so at 10x/100x invocation load semantic mode will fail or degrade unpredictably.
- Evidence: `"POST to localhost:11434/api/embeddings"` and `curl http://localhost:11434/api/embeddings`; `"If --semantic and Ollama available ... Else: fall back to keyword search"`.

**[HIGH] Query Path Performs Index Maintenance, Creating Latency Spikes and Throughput Collapse**
- Location: Phase 2 Search flow
- Issue: Running cache freshness checks and re-embedding inside request handling couples query latency to indexing work.
- Evidence: `"If --semantic ... Check cache freshness ... Re-embed stale files (incremental) ... Embed query ... Return top-5"`.
- Recommendation: Split into read path vs write path: background indexer updates snapshots, query path reads last committed snapshot only.

**[HIGH] JSON Cache File Is Shared Mutable State and a Concurrency Bottleneck**
- Location: Phase 1 Cache management; Error Handling
- Issue: A single JSON cache file will race under concurrent processes, and full-file rewrites do not scale with dataset growth.
- Evidence: `"Cache management — logs/cache/vault_embeddings.json"`; `"save_embedding_cache(cache: Dict): # Atomic write to cache file"`; `"Cache corrupt | Rebuild from scratch"`.
- Recommendation: Replace JSON cache with SQLite (WAL) or an embedded KV store keyed by `(path, mtime, model, schema_version)` and guarded by process locks/singleflight.

**[HIGH] Model/Vector Contract Is Implicit and Fragile**
- Location: Phase 1 Embed function; Implementation Details; Cache schema
- Issue: The plan hardcodes model and 768-dim assumptions without versioned cache compatibility, so model swaps can poison results or break cosine comparisons.

- Recommendation: Version the cache with `model_id`, `embedding_dim`, `extractor_version`, and reject/rehydrate incompatible entries automatically.

**[HIGH] Scalability Validation Is Missing Critical Throughput and Latency Gates**

### Findings

**[HIGH] Cache schema lacks backward-compatible versioning**
- Location: Action Items → Phase 1: Cache management; Deliverables
- Issue: Cache format is unversioned, so rolling back code can break on incompatible cache structure.
- Evidence: `"Returns {filepath: {"embedding": [...], "mtime": ...}}"` and `"Cache location: logs/cache/vault_embeddings.json"`.

**[HIGH] Full refresh has no last-known-good restore path**
- Location: Action Items → Phase 3; Verification → Test Results
- Issue: A failed or low-quality `--refresh-cache` run can replace healthy embeddings without an operational restore mechanism.
- Evidence: `"--refresh-cache flag forces full re-embedding"` and `"Cache refreshed: 99 embeddings saved."`

**[HIGH] Output contract change can cascade to downstream tooling**
- Location: Action Items → Phase 2
- Issue: Hybrid output changes may break consumers expecting prior keyword-only output shape.
- Evidence: `"JSON output includes both semantic and keyword results"` and `"--semantic flag triggers semantic + keyword hybrid search"`.

**[MEDIUM] Fallback behavior is graceful but operationally silent**

- Issue: Keyword fallback on semantic failures can mask prolonged semantic outages.
- Evidence: `"Ollama not running | Log warning, fallback to keyword"` and target `"Zero crashes when Ollama unavailable"`.
- Recommendation: Emit metrics for fallback frequency and error type; gate releases/alerts when fallback exceeds a threshold.

**[MEDIUM] Feature gating is caller-controlled, not operator-controlled**
- Location: Action Items → Phase 2; Notes
- Issue: `--semantic` is not a central kill switch and becomes insufficient if semantic is later defaulted or automated.
- Evidence: `"Add --semantic flag"` and `"Consider making semantic the default if evaluation proves superiority"`.
- Recommendation: Add an operator kill switch (env/config flag) that can disable semantic path without redeploy, plus a cleanup plan.

**[MEDIUM] Partial indexing risk is unbounded**
- Location: Error Handling; Implementation Details
- Issue: Skipping failed embeddings can produce unstable retrieval quality with no minimum integrity gate.
- Evidence: `"Embedding fails | Skip file, continue with others"` and `"Embed these fields from each learning: Title ... Thesis ... Tags"`.
- Recommendation: Enforce minimum embedding coverage and auto-disable semantic results below threshold.

**[LOW] Validation omits rollback drill and recovery SLO**
- Location: Evaluation Criteria; Success Metrics
- Issue: Plan can pass validation without proving rollback works under fault.
- Evidence: `"Fallback reliability | Stop Ollama mid-search | Graceful keyword fallback"` and no rollback metric in success checklist.
- Recommendation: Add a rollback game-day test with target recovery time and mandatory evidence artifact.

### Top 3 Recommendations

1. Add a phase-by-phase rollback runbook with explicit irreversible checkpoints and timed recovery validation (Finding: **[CRITICAL] No phase-level rollback procedure for refactored search path**).
2. Version the embedding cache format and support dual-read compatibility before any rollout (Finding: **[HIGH] Cache schema lacks backward-compatible versioning**).

## Coupling Detector (opencode)

### Findings

**[HIGH] Hidden Cache Format Coupling**
- Location: cto_context_lib.py lines 491-527, cto_context_search.py lines 44-71
- Issue: Cache format (`{filepath: {"embedding": [...], "mtime": ...}}`) is an implicit contract between 5 functions; changing it requires coordinated changes across `load_embedding_cache`, `save_embedding_cache`, `get_stale_files`, `search_semantic`, and the `--refresh-cache` implementation
- Evidence: cto_context_search.py:61-65 manually constructs cache entries with the same structure as cto_context_lib.py:679-682
- Recommendation: Create a `CacheEntry` dataclass with serialization methods to make the contract explicit

**[HIGH] Temporal Coupling in Ollama Availability Check**
- Location: cto_context_search.py lines 40-71, 76-86
- Issue: `--refresh-cache` performs early Ollama check that duplicates logic in `search_semantic`, creating two paths that must stay synchronized; if `ollama_available` changes its check logic, `--refresh-cache` will behave differently than normal search
- Evidence: Line 41 checks `ollama_available()` and exits, but `search_semantic` at line 657-658 does its own check and returns empty list
- Recommendation: Remove early exit from `--refresh-cache` and let it use the same graceful fallback pattern as normal search

**[MEDIUM] Conditional Import Creates Hidden Dependency**
- Location: cto_context_search.py lines 44-47
- Issue: Functions imported conditionally inside `if '--refresh-cache'` block (`load_embedding_cache`, `save_embedding_cache`, `_extract_learning_text`, `embed_text`, `ROOT`) create a second import surface that must be manually maintained
- Evidence: If `cto_context_lib.py` renames `embed_text`, the `--refresh-cache` code breaks while normal imports work
- Recommendation: Move all imports to module level or extract `--refresh-cache` to a separate script

**[MEDIUM] Hardcoded Model Defaults Create Cross-Module Coupling**
- Location: cto_context_lib.py line 463, cto_context_search.py lines 81-84
- Issue: `DEFAULT_EMBED_MODEL = "nomic-embed-text"` in lib is used by search, but if the model changes, cached embeddings become invalid without cache version bump
- Evidence: No version field in cache format; changing model would silently return stale/incorrect similarity scores
- Recommendation: Include model name and version in cache entry; validate on load

**[MEDIUM] Shared Global State for Cache Directory**
- Location: cto_context_lib.py lines 461-462
- Issue: `CACHE_DIR` and `EMBEDDING_CACHE_FILE` are module-level globals that create ambient coupling; tests cannot easily override
- Evidence: Functions call `_ensure_cache_dir()` which uses global `CACHE_DIR`; no dependency injection path
- Recommendation: Pass cache path as parameter with default, or use a configuration object

**[LOW] Duplicate File Discovery Logic**
- Location: cto_context_lib.py lines 665-666, cto_context_search.py lines 52
- Issue: Learning file discovery logic repeated in two places with slight differences (transcript inclusion)
- Evidence: `learning_files = [f for f in learnings_path.glob("*.md") if not f.name.startswith('_')]` appears in both files
- Recommendation: Extract `get_learning_files(include_transcripts=False)` function to lib

### Top 3 Recommendations

1. **Create explicit CacheEntry dataclass with model versioning** — addresses HIGH cache format coupling and MEDIUM model default coupling
2. **Remove conditional imports and either move all to top or extract `--refresh-cache` to separate script** — addresses HIGH temporal coupling and MEDIUM hidden dependency
3. **Add cache format version field and validation** — prevents silent failures when model or format changes
