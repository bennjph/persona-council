  2. Define strict behavioral contracts for fallback/hybrid/output/performance (exit codes, schemas, merge rules, timing preconditions) so every requirement has binary pass/fail checks (Findings: Fallback Contract, Hybrid Interaction, Performance Preconditions, Output Inconsistency).
  3. Add explicit test-data fixtures and automation classification for all non-happy paths (corrupt cache, model missing, embedding failure, TTL override) and state CI feasibility per test (Findings: Failure-Path Data Missing, TTL Not Verifiable, Automation Feasibility Missing).

**Edge Case Hunter:**
  1. **Implement path validation for all cache operations** — Use `Path.resolve()` and verify paths remain within ROOT before file operations; this prevents the most severe attack vector (arbitrary file access via cache poisoning)
  2. **Add file locking and atomic cache updates** — Use `filelock` library to ensure only one process modifies cache at a time; prevents corruption under concurrent access
  3. **Enforce resource limits with graceful degradation** — Add `--max-files`, `--max-file-size` limits; skip oversized files with warning rather than attempting to embed them; prevents DoS from unbounded processing
