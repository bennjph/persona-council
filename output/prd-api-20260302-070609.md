- Evidence: “Return 768-dim vector” and `curl http://localhost:11434/api/embeddings ... "embedding": [0.023, -0.145, ...]`

- Issue: Shared cache reads/writes are planned but no locking/transaction semantics are required, creating race-condition risk between search and refresh.
- Evidence: “Background refresh option — `python3 scripts/cto_context_search.py --refresh-cache`” and “Smart caching — Only re-embed files modified since last cache”

**[HIGH] Ambiguous "Embedding Target" leads to inconsistent vector generation**
- Location: Implementation Details / Embedding Target
- Issue: The instruction to concatenate "title + ' | ' + thesis + ' | ' + tags" does not define behavior when fields are missing (e.g., no tags, no clear thesis paragraph). This will produce different embedding inputs for structurally different markdown files.

**[MEDIUM] Boundary condition for "score" calculation and ranking is undefined**

- Recommendation: Specify that `load_embedding_cache` should return an empty dict on any JSON decode error, file not found, or schema validation failure, triggering a silent rebuild on next embedding operation.

**[LOW] Ambiguous "Verification" success criteria**
- Location: Verification / Test Results
- Issue: The success metric "Semantic search finds ≥3 relevant learnings that keyword search misses" is demonstrated with one example query. There is no definition of "relevant" or a controlled test set, making verification subjective.

### Top 3 Recommendations

1. **Define failure mode contracts for Ollama API calls** (relates to CRITICAL finding). Specify HTTP timeout (e.g., 5s), retry logic (e.g., 0 retries), and exact conditions that trigger fallback to keyword search, ensuring deterministic behavior.

## Edge Case Hunter (qwen-3.5-27b)

### Findings

**[CRITICAL] Path Traversal in Backup Function**
- Location: `scripts/cto_context_lib.py` - `backup_before_write` function
- Issue: The function accepts any filepath string without validation, allowing attackers to backup arbitrary files outside the intended vault directory.
- Evidence: `def backup_before_write(filepath: str | Path) -> Optional[Path]:` with no path sanitization or root directory validation
- Recommendation: Add `src.resolve().is_relative_to(ROOT)` check before backup; reject paths that escape ROOT directory

**[CRITICAL] No Input Length Limits on Query Parameter**
- Location: `scripts/cto_context_search.py` - `--query` argument
- Issue: Query strings have no maximum length, enabling resource exhaustion through extremely long queries that could cause memory exhaustion or Ollama API timeouts.
- Evidence: `parser.add_argument("--query", required=True, help="Query string.")` with no `max_length` or validation
- Recommendation: Add `max_length=1000` validation on query; reject queries exceeding limit with clear error message

**[HIGH] Ollama API Calls Lack Timeout Configuration**
- Location: `scripts/cto_context_lib.py` - `embed_text` function (implied)
- Issue: HTTP requests to Ollama have no timeout, causing indefinite hangs if Ollama becomes unresponsive.
- Evidence: Spec shows `POST to localhost:11434/api/embeddings` but no timeout parameter specified
- Recommendation: Add `timeout=30` seconds to all urllib requests; implement retry with exponential backoff (max 3 attempts)

**[HIGH] Cache File Has No Atomic Write Guarantee**
- Location: `logs/cache/vault_embeddings.json` - `save_embedding_cache` function
- Issue: Spec states "Atomic write to cache file" but implementation not shown; partial writes during crash could corrupt entire cache.
- Evidence: `def save_embedding_cache(cache: Dict): # Atomic write to cache file` with no implementation details
- Recommendation: Implement write-to-temp-then-rename pattern; add JSON validation on load with fallback to rebuild

**[HIGH] No Validation on Embedding Cache Integrity**
- Location: `logs/cache/vault_embeddings.json` - cache structure
- Issue: Cache stores embeddings without checksums; corrupted or tampered embeddings could return malicious search results.
- Evidence: Cache structure shows `{"embedding": [...], "mtime": ...}` with no hash or validation field
- Recommendation: Add SHA-256 hash of file content to cache; verify hash before using cached embedding

**[MEDIUM] Concurrent Cache Access Not Protected**
- Location: `logs/cache/vault_embeddings.json` - cache file
- Issue: Multiple search processes could read/write cache simultaneously, causing race conditions and corruption.
- Evidence: No file locking mechanism mentioned in cache management functions
- Recommendation: Add file locking (fcntl on Unix) during cache read/write; or use atomic operations with temp files

**[MEDIUM] No Maximum Limit on Search Results**
- Location: `scripts/cto_context_search.py` - `--limit` argument
- Issue: Limit parameter has no maximum, allowing resource exhaustion through large result sets.
- Evidence: `parser.add_argument("--limit", type=int, default=15, help="Max matches.")` with no upper bound

**[MEDIUM] Empty Query Not Handled in Semantic Search**

- Issue: Empty or whitespace-only queries could cause Ollama API errors or return all files.
- Evidence: `tokens = [t.lower() for t in query.strip().split() if t.strip()]` returns empty list but no explicit error
- Recommendation: Return error for empty queries before calling Ollama API; don't allow empty semantic searches

**[MEDIUM] Backup Directory Not Validated**
- Location: `scripts/cto_context_lib.py` - `BACKUP_DIR` constant
- Issue: Backup directory path is hardcoded but not validated for writability or existence before use.
- Evidence: `BACKUP_DIR = ROOT / "logs" / "backups"` with `BACKUP_DIR.mkdir(parents=True, exist_ok=True)` but no error handling
- Recommendation: Check `BACKUP_DIR.is_dir()` and writability before backup; fail gracefully with clear error if backup impossible

**[LOW] No Cache Size Limit or Pruning Strategy**
- Location: `logs/cache/vault_embeddings.json` - cache file
- Issue: Cache grows indefinitely; no maximum size or pruning of old entries mentioned.
- Evidence: "Cache grows too large" listed as risk with "Compress embeddings; prune old files" mitigation but no implementation

**[LOW] Model Name Not Validated**

- Evidence: `def embed_text(text: str, model: str = "nomic-embed-text")` with no model whitelist

### Top 3 Recommendations

1. **Add path traversal protection to `backup_before_write`** (relates to CRITICAL finding #1) - Implement `src.resolve().is_relative_to(ROOT)` check to prevent arbitrary file backup/overwrite attacks
